# Regras obrigatórias do projeto

## Processo

- Trabalhar uma fase por vez.
- Não desenvolver múltiplas seções grandes em uma única solicitação.
- Não alterar áreas já aprovadas sem autorização.
- Utilizar apenas os arquivos necessários em cada tarefa.
- Informar impactos antes de realizar refatorações estruturais.
- Preservar decisões validadas.
- Criar resumo de continuidade ao finalizar cada fase.

## Conteúdo

- Não inventar métricas.
- Não inventar depoimentos.
- Não inventar clientes.
- Não inventar resultados financeiros.
- Não apresentar Ammis Moda como projeto criado integralmente pela Dreh Work.
- Não sugerir que existe uma grande equipe.
- Não utilizar informações pessoais desnecessárias.

## Design

- Base visual preta e branca.
- Uma cor estratégica de destaque.
- Estética futurista, moderna, criativa e tecnológica.
- Utilizar Poppins como tipografia principal.
- Evitar aparência de template.
- Evitar estética genérica de inteligência artificial.
- Evitar excesso de neon, glassmorphism e partículas.
- Utilizar animações intensas, mas controladas.
- Respeitar prefers-reduced-motion.
- Priorizar conversão e clareza.

## Desenvolvimento

- Utilizar TypeScript.
- Priorizar desempenho.
- Priorizar acessibilidade.
- Componentizar adequadamente.
- Evitar dependências desnecessárias.
- Manter estrutura simples de atualizar.
- Preparar portfólio para receber novos projetos.
- Preparar integrações futuras sem criar complexidade prematura.
- Não incluir segredos ou credenciais no repositório.
